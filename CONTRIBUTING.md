The following is a set of guidelines for contributing to this project. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## How Can I Contribute?

### Reporting Bugs

Start by searching GitHub Issues to check if the problem has already been reported. If not, please open a new issue and include:
- A clear title and description.
- Use cases or steps to reproduce.
- Screenshots if relevant.

### Suggesting Enhancements

Enhancement suggestions are tracked as GitHub Issues. create an issue and:
- Use a clear and descriptive title.
- Provide a step-by-step description of the suggested enhancement.
- Explain why this enhancement would be useful.

### Pull Requests

1.  Fork the repo and create your branch from `main`.
2.  If you've added code that should be tested, add tests.
3.  Ensure your code lints and builds (`npm run build`).
4.  Issue that pull request!

## Styleguides

### Git Commit Messages

- Use the present tense ("Add feature" not "Added feature")
- Use the imperative mood ("Move cursor to..." not "Moves cursor to...")
- Limit the first line to 72 characters or less

### Javascript Styleguide

- We use ESLint and Prettier.
- Run `npm run lint` before committing.

Thank you for contributing!
